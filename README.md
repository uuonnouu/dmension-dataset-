# Dmension Mathematical Universe
### Open Dataset - Parametric 3D Model Library

> **The models are artifacts. The engine is the invention.**  
> This repository publishes outputs of the **Dmension Mathematical Universe** generative system, a proprietary mathematical engine that builds continuous-topology 3D geometry from pure parametric functions. The source engine is not included. What you're testing against is its output.

---

## What This Is

A curated collection of 3D models generated by the Dmension engine: real-valued XYZ meshes with full UV mapping, continuous manifold topology, embedded normals, and in several cases, baked animation and lighting data. Formats: **GLB, OBJ, USDZ**, with accompanying render images.

These are not scanned objects. They are not photogrammetry. They are not AI-generated meshes.  
They are **mathematical constructions**, outputs of a deterministic parametric system with tunable geometric parameters, exported at various resolution and complexity levels.

**Total dataset size:** ~50 GB across formats.

---

## Why This Exists

Most public 3D datasets suffer from one or more of:
- Broken topology (non-manifold edges, T-junctions, degenerate faces)
- Missing or invalid UV coordinates
- Static meshes with no animation or deformation data
- Unknown or inconsistent generation provenance

This dataset provides the opposite: clean manifold topology, full UV (TEXCOORD_0), normals, and in animated models, keyframe data with named animation tracks. Every model has a known mathematical origin.

**Intended uses (see License):**
- AI/ML training and benchmarking
- UV unwrapping algorithm development
- 3D reconstruction ground-truth evaluation
- AR/WebXR deployment testing (GLB + USDZ dual-format)
- Generative model comparison baselines
- Academic geometry processing research

## Example Model: `foundation_tensor_mesh_ar_universal`

The file `foundation_tensor_mesh_ar_universal.glb` is the reference specimen for this dataset.

| Property | Value |
|---|---|
| Format | GLB (glTF 2.0 binary) |
| Generator | THREE.GLTFExporter (Dmension engine → Three.js export pipeline) |
| Vertices | 5,658 |
| Triangles | 10,980 |
| UV Sets | 1 (TEXCOORD_0, full [0,1] range) |
| Normals | Embedded (per-vertex) |
| Bounding Box | ~4 × ~4 × ~4 units |
| Animation | `ARPulse` — scale channel, 60 keyframes, 10-second loop |
| Lighting | KHR_lights_punctual (directional, intensity 0.8) |
| Material | KHR_materials_emissive_strength |
| Extensions | `KHR_lights_punctual`, `KHR_materials_emissive_strength` |
| Topology | Continuous manifold — no non-manifold edges |

The `ARPulse` animation drives uniform scale oscillation (0.9×–1.1×) over 10 seconds — designed for AR breathing/presence effects in ARKit and WebXR without skeletal rigging overhead.

**To view immediately:** Drag the `.glb` into [gltf.report](https://gltf.report) or [model-viewer.dev](https://modelviewer.dev/editor/).  
**iOS AR:** Open the `.usdz` on any iPhone/iPad — ARKit Quick Look launches automatically.

## The Engine: Dmension Mathematical Universe

The models here are outputs. The engine that built them is a separate proprietary system, the **Dmension Mathematical Universe** — developed under the UUON Foundation.

It is a mathematical construction engine, not a modeler. It does not sculpt. It does not import. It derives geometry from:

- Continuous parametric functions over XYZ space
- UV coordinate generation from the same mathematical basis (not post-process unwrapping)
- Multi-parameter deformation pipelines (scale, curvature, field interaction, topological genus)
- Deterministic export at any resolution without re-baking

This means UV seams are not artist decisions,they are mathematical boundaries. Normal directions are not smoothed heuristics, they are derived surface normals. Topology is not retopologized, it is generated correct.

The engine is not open source. The dataset it produces is.
